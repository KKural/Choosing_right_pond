Replication package for the BJC manuscript
Choosing the right pond: Spatial clustering of street graffiti by visual complexity

This package contains the data, scripts, and generated outputs needed to reproduce the analysis.

Package contents

1) Root folder files
- BJC_Choosing_right_pond_rep_pkg.Rproj
- README.txt

2) Data folder
Path: Data
Files:
- Appendix 1.docx
- df_graffiti_items_anonymized.csv
- WBN3.shp
- WBN3.shx
- WBN3.dbf
- WBN3.prj
- WBN3.qpj
- BRUGGEN 2.shp
- BRUGGEN 2.shx
- BRUGGEN 2.dbf
- BRUGGEN 2.prj
- BRUGGEN 2.qpj

3) Scripts folder
Path: Scripts
Files:
- Choosing_right_pond_functions.R
- Choosing_right_pond_reproduce.R

4) Output folder (generated by running the script)
Path pattern: YYYYMMDD_script_choosing_right_pond_outputs

How to use this package

1. Open BJC_Choosing_right_pond_rep_pkg.Rproj in RStudio.
2. The RStudio project automatically sets the current folder as the working directory.
3. Open Choosing_right_pond_reproduce.R from the Scripts folder and run it.
4. Wait for the script to finish. It writes all analysis outputs to a dated output folder.
5. The first run may take some time. From the second run onward, permutation results are saved in the cache folder inside the output folder and reused, so runtime is shorter.
6. The output folder contains all required files (Excel, figure, metadata, and appendix documents).

Required R packages
dplyr, tidyr, readxl, writexl, sf, ggplot2, ggspatial, patchwork, vegan,
flextable, officer, janitor, extrafont, here, and other dependencies loaded by the scripts.

Reproducibility note
If the output folder for the current date already exists, the script reuses that folder name and updates files.

IMPORTANT: Keep folder structure and names unchanged
Do not rename, move, or delete folders or files in Data and Scripts.
The scripts use fixed relative paths and will fail if names or structure are changed.
